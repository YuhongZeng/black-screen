# 假设驱动复现矩阵

## 先纠正一点

内存压力、working set trim、打开网页都不是“我们认为加了就一定复现”的根因。它们只是加速器，用来验证一个更具体的假设：

> IDE 被遮挡/隐藏后，Chromium/Viz 的 surface、software output 或 frame sink 进入陈旧/被回收状态；恢复 visible 时状态同步没有完整完成。

所以复现设计必须每轮只验证一个断点，而不是把所有压力都堆上去。

## 当前最高怀疑

最高怀疑是 Chromium/Electron 的显示状态同步链路：

```text
visible
  -> occluded / hidden
  -> paint/rAF/BeginFrame 降级或暂停
  -> surface/frame/software output 陈旧或被回收
  -> restore/show
  -> WasShown / SetVisible / LocalSurfaceId / FrameSink / Viz 未完全同步
  -> IDE 顶层窗口黑屏或显示旧画面
```

插件可能只是触发复杂 surface 拓扑：

- 两个 Webview。
- Webview 隐藏期间仍有状态变化。
- 真实 agent 生成代码造成 editor/workbench 变化。
- agent 完成后，复杂 Webview/workbench surface 状态仍保留。

## 三类状态要分开测

### A. Agent 活跃输出时复现

目的：验证“生成代码过程中”是否是必要条件。

```powershell
.\scripts\repro-episode.ps1 `
  -ProcessName Code `
  -AgentPromptDir .\agent-prompts `
  -AgentFocusKeys "^+a" `
  -PromptBeforeEachLoop `
  -AgentRunSeconds 90 `
  -Loops 10 `
  -CoverSeconds 180
```

### B. Agent 已经结束后复现

目的：验证 agent 只负责留下复杂 surface 状态，真正触发发生在之后。

```powershell
.\scripts\repro-episode.ps1 `
  -ProcessName Code `
  -AgentPromptDir .\agent-prompts `
  -AgentFocusKeys "^+a" `
  -PromptBeforeEachLoop `
  -AgentRunSeconds 90 `
  -AgentCooldownSeconds 180 `
  -Loops 10 `
  -CoverSeconds 180
```

如果 B 能复现，说明“agent 正在生成代码”不是必要条件。

### C. Agent 已结束 + 用户打开网页/做别的事

目的：模拟现场“agent 跑完后打开网页也复现”。

```powershell
.\scripts\repro-episode.ps1 `
  -ProcessName Code `
  -AgentPromptDir .\agent-prompts `
  -AgentFocusKeys "^+a" `
  -PromptBeforeEachLoop `
  -AgentRunSeconds 90 `
  -AgentCooldownSeconds 180 `
  -Loops 10 `
  -CoverSeconds 180 `
  -OpenLocalStressPage
```

本地压力页在：

```text
web-stress/stress.html
```

它会制造浏览器动画、合成层和内存活动，用来模拟打开网页对 DWM/GPU/内存的扰动。

## 加速器的意义

### Working set trim

验证 surface/shared bitmap/software output 是否在后台资源回收后更容易不同步。

```powershell
-TrimWorkingSet
```

如果加它明显提高复现率，说明“后台资源被回收后 restore 漏重建”的方向更强。

### 内存压力

验证系统资源压力是否促使隐藏窗口 surface 更早被 evict。

```powershell
-MemoryPressureMb 4096 -MemoryPressureSeconds 60
```

如果只有内存压力下复现，不能直接说根因是内存；只能说明 eviction/资源回收是触发器。

### 小窗口遮挡 vs 最小化

小窗口遮挡对应 occluded 路径。

```powershell
# 默认就是小窗口遮挡
```

最小化对应 hidden 路径。

```powershell
-UseMinimize
```

如果小窗口遮挡就能复现，说明不只是 `SC_MINIMIZE`，普通 occlusion 也能破坏状态同步。

## 提示词自动投喂

现在支持提示词目录轮询：

```powershell
-AgentPromptDir .\agent-prompts
```

目录下的 prompt 会按轮次循环使用：

- `01-generate-feature.txt`
- `02-refactor-and-test.txt`
- `03-build-ui-or-page.txt`
- `04-fix-and-iterate.txt`

如果你们没有 Agent 输入框快捷键，用相对主窗口坐标：

```powershell
-AgentClickX 1450 -AgentClickY 980
```

如果有快捷键，例如 `Ctrl+Shift+A`：

```powershell
-AgentFocusKeys "^+a"
```

## 建议的第一轮实验

1. A：活跃输出 + 小窗口遮挡，无压力。
2. B：已结束 + 小窗口遮挡，无压力。
3. C：已结束 + 小窗口遮挡 + 打开本地压力页。
4. B + `-TrimWorkingSet`。
5. C + `-TrimWorkingSet`。

只有当这些都不复现时，再加内存压力。

这样能回答：

- agent 是否必须正在工作。
- occlusion 是否足够。
- 用户打开网页是否是有效加速器。
- resource eviction 是否是有效加速器。
