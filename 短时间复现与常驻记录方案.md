# IDE Windows 黑屏：短时间复现与常驻记录方案

目标不是只做最大化/最小化循环，而是构造更接近现场的 episode：

1. 真实 agent/插件负载先运行一段时间。
2. IDE 被小窗口遮挡、最小化或放后台。
3. 后台期间叠加 working set trim、内存压力、显示状态变化或用户打开网页等扰动。
4. 恢复 IDE 后自动截图判黑。
5. 命中后保留截图、进程信息、trace、hook 日志。

## 为什么要用小窗口遮挡

现场不只最小化会复现，被小窗口遮挡也会复现，且有时 agent 已经跑完，之后用户打开网页/做别的事情才复现。这说明触发条件可能是：

- IDE 窗口长期处于后台/遮挡。
- Chromium 将 Webview/workbench 标记为 hidden/occluded，停止 paint/rAF。
- 后台期间 DWM/Chromium/Viz 回收或失效 surface。
- 恢复时父 workbench surface 或 Webview 子 surface 没正确重新挂回。

因此复现要覆盖遮挡，不应只覆盖最小化。

## 推荐先跑的 4 个 episode

所有实验都使用真实有问题的 IDE 和真实 agent 插件。

### E1：小窗口遮挡

```powershell
cd G:\crash
.\scripts\repro-episode.ps1 -ProcessName Code -Loops 10 -WarmupSeconds 60 -CoverSeconds 300
```

### E2：小窗口遮挡 + working set trim

```powershell
.\scripts\repro-episode.ps1 -ProcessName Code -Loops 10 -WarmupSeconds 60 -CoverSeconds 180 -TrimWorkingSet
```

### E3：小窗口遮挡 + 内存压力

```powershell
.\scripts\repro-episode.ps1 -ProcessName Code -Loops 10 -WarmupSeconds 60 -CoverSeconds 180 -MemoryPressureMb 4096 -MemoryPressureSeconds 60
```

根据机器内存调整 `MemoryPressureMb`，不要第一次就吃满。

### E4：最小化 + working set trim + 内存压力

```powershell
.\scripts\repro-episode.ps1 -ProcessName Code -Loops 10 -WarmupSeconds 60 -CoverSeconds 180 -UseMinimize -TrimWorkingSet -MemoryPressureMb 4096 -MemoryPressureSeconds 60
```

## 单独工具

小窗口遮挡：

```powershell
.\scripts\cover-window.ps1 -ProcessName Code -Seconds 300
```

Trim 所有 IDE 相关进程 working set：

```powershell
.\scripts\trim-working-set.ps1 -ProcessName Code
```

制造内存压力：

```powershell
.\scripts\memory-pressure.ps1 -Megabytes 4096 -HoldSeconds 60
```

截图判黑：

```powershell
.\scripts\test-window-black.ps1 -ProcessName Code
```

## 判定结果

`repro-episode.ps1` 每轮会写入：

```text
.black-screen-repro/episodes/episode_xxx/
  window-shot.png
  black-check.json
  trim-working-set.txt
  processes.json    # 仅命中疑似黑屏时写
```

如果 `black-check.json` 中 `isBlack=true`，脚本会停止，保留现场。

## 下一步优化

如果 E2/E3/E4 能明显缩短复现时间，再把 WinDbg 低频日志断点和 Perfetto trace 放进同一个 episode runner 中，让它命中后自动保留 trace/dump/log。
