# 大模型状态判断说明

## 已知 UI 规则

当前不是“输入框无法输入”，而是：

- 某个会话输入后返回错误。
- 如果输出以 `Cannot connect to API` 开头，就代表该会话要废弃，需要新建会话。
- 新建会话后的输入框位置固定。
- 另一个输入框位置也固定。

## 外部脚本能判断什么

不接入插件内部状态时，外部脚本不能可靠读取 Electron/Webview 里的文本内容。要自动识别 `Cannot connect to API`，需要以下任一条件：

1. 插件开发提供状态日志或消息事件。
2. Webview/DOM 探针把最后一条 assistant 输出写到文件。
3. 使用 OCR 识别截图区域。
4. Electron accessibility tree 能稳定读到文本。

目前我们先不依赖这些不稳定方式。

## 更稳的策略

既然错误会话需要新建，而新建会话入口和新输入框位置固定，复现 runner 可以每轮都执行：

```text
点击新会话/打开会话固定点
点击新输入框固定点
粘贴提示词
回车
```

这样不需要判断上一轮会话是否已经 `Cannot connect to API`。

## 怎么判断模型是否结束

外部无侵入方案不能准确知道“大模型内部是否结束”，但可以用视觉稳定近似判断：

- 监控 Agent 输出区域截图。
- 如果连续 N 秒区域变化很小，就认为输出结束或卡住。
- 如果持续变化，就认为仍在输出。

新增脚本：

```powershell
.\scripts\wait-agent-visual-stable.ps1
```

单独测试示例：

```powershell
.\scripts\wait-agent-visual-stable.ps1 `
  -ProcessName codeArts-agent `
  -RegionX 780 `
  -RegionY 160 `
  -RegionWidth 1050 `
  -RegionHeight 760 `
  -MaxSeconds 180 `
  -QuietSeconds 20
```

坐标同样是相对 IDE 主窗口左上角。区域应该覆盖 Agent 输出区域，不要覆盖输入框和频繁闪烁的光标。

## 在 stable runner 中使用视觉稳定等待

```powershell
.\scripts\repro-stable-runner.ps1 `
  -ProcessName codeArts-agent `
  -WindowMode cover `
  -RestoreMode maximize `
  -Episodes 10 `
  -AgentPromptDir .\agent-prompts `
  -AgentClickSequence "120,160;1450,980" `
  -PromptEachEpisode `
  -AgentWaitMode visualStable `
  -AgentOutputX 780 `
  -AgentOutputY 160 `
  -AgentOutputWidth 1050 `
  -AgentOutputHeight 760 `
  -AgentVisualMaxSeconds 180 `
  -AgentVisualQuietSeconds 20 `
  -AgentCooldownSeconds 120 `
  -HiddenSeconds 180 `
  -PostRestoreObserveSeconds 600 `
  -PostRestoreCheckIntervalSeconds 5
```

如果视觉稳定等到了，会进入后续遮挡/恢复步骤。

## 如果必须识别 Cannot connect to API

最可靠方案是请插件开发加一个轻量探针：

```text
每当 assistant 输出第一行变化时，写入 workspace 日志：
agent-status.ndjson
字段：time, conversationId, firstLine, isGenerating, isError
```

其中：

```text
isError = firstLine startsWith "Cannot connect to API"
```

没有这个探针时，外部脚本只能通过 OCR 或视觉变化猜测，不建议作为自动化判据。

## 如果输出文本区域可复制

如果 Agent 输出日志区域位置固定，并且里面的文字可以通过鼠标拖选后 `Ctrl+C` 复制，可以用外部脚本做一个较可靠的检查：

```powershell
.\scripts\read-agent-output-text.ps1 `
  -ProcessName codeArts-agent `
  -StartX 780 `
  -StartY 160 `
  -EndX 1600 `
  -EndY 260 `
  -DetectPrefix "Cannot connect to API" `
  -OutFile .\.black-screen-repro\agent-output-check.json
```

坐标是相对 IDE 主窗口左上角：

- `StartX/StartY`：从错误输出第一行左上附近开始拖选。
- `EndX/EndY`：拖到能覆盖第一行或前几行的位置。

脚本会：

1. 将 IDE 置前。
2. 从起点拖选到终点。
3. 发送 `Ctrl+C`。
4. 读取剪贴板文本。
5. 判断文本是否以 `Cannot connect to API` 开头。

如果匹配，脚本退出码为 `2`，并输出：

```json
{"isMatch":true,"firstLine":"Cannot connect to API..."}
```

注意：

- 这会改变当前选择区域，属于有轻微 UI 干扰的检查，不建议在黑屏现场命中后使用。
- 如果 Webview 输出文本不可选择，或者拖选坐标不准，剪贴板会为空或不是目标文本。
- 先单独用这个脚本校准坐标，确认能稳定复制第一行，再考虑接入 runner。
